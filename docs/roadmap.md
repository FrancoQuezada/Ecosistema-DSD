# Roadmap

## MVP actual

- Sitio público institucional.
- Banco de desafíos con datos mock.
- Formulario de postulación con confirmación y almacenamiento local.
- Portafolio mock de soluciones digitales.
- Panel interno simple para revisar desafíos y puntajes.
- Tipos TypeScript para las entidades centrales.
- Helpers de scoring ponderado para evaluación de desafíos y continuidad.

## Próxima iteración

1. Persistencia real de desafíos en Supabase o PostgreSQL.
2. Validación de formularios con esquema compartido.
3. Autenticación institucional para `/admin/desafios`.
4. Vista detalle de desafío.
5. Flujo de evaluación con rúbricas editables.
6. Estados de transición: postulado, revisión, evaluado, seleccionado, proyecto, cierre.
7. Registro de proyectos derivados de desafíos seleccionados.

## Iteración de continuidad

1. Módulo de proyectos con equipo, sponsor, entregables y validaciones.
2. Evaluación de continuidad con criterios habilitantes.
3. Decisión de laboratorio, piloto, adopción, transferencia o cierre documentado.
4. Generación de ficha de cierre del proyecto.
5. Publicación controlada en portafolio.

## Iteración institucional

1. Roles y permisos: estudiante, académico, comité, unidad interna, socio externo.
2. Métricas de impacto y seguimiento semestral.
3. Integración con repositorios y demos.
4. Catálogo histórico de soluciones y aprendizajes.
5. Reportes para docencia, investigación aplicada y vinculación.
